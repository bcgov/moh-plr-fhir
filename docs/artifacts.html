This page provides a list of the FHIR artifacts defined as part of this implementation guide.

Structures: Resource Profiles

These define constraints on FHIR resources for systems conforming to this implementation guide

BCLocation

General constraints on the Location resource for use in the BC Provider Registry project.
BCOrganization

General constraints on the Organization resource for use in the BC Provider Registry project.
BCOrganizationAffiliation

General constraints on the OrganizationAffiliation resource for use in the BC Provider Registry project.
BCPractitioner

General constraints on the Practitioner resource for use in the BC Provider Registry project.
BCPractitionerRole

General constraints on the PractitionerRole resource for use in the BC Provider Registry project.

Structures: Data Type Profiles

These define constraints on FHIR data types for systems conforming to this implementation guide

BCAddress

Adds a validity flag and a status to the address.
BCContactPoint

Adds a status, validity flag and extra contact information.
BCHumanName

Adds a status, validity flag and period, and a use code to a name.
BCIdentifier

Adds a status to the identifier.

Structures: Extension Definitions

These define constraints on FHIR data types for systems conforming to this implementation guide

BC Extra birthdate information

Adds birth time and a period to the birth date.
BC Birth Place Extension

Captures the place where a patient/practitioner/person was born.
BC Facility Community Health Service Area

A code that represents the community health service area that the location is a part of.
BC Practitioner Date of Death Extension

The Date of Death of a Practitioner.
BC Facility Details Extension

Provides the details of a facility as represented by a FHIR location.
BC License Status Extension

Tracking the status and changes to the status of a practitioner/organization license.
BC Facility Mailing Address

An optional mailing address for the facility.
BC Name Use Extension

Allows for use on a simple name string.
BC Notes

Notes about the practitioner/location/organization.
BC Valid Period

The period for when the extended element is valid.
BC Practitioner Conditions

Conditions on the provider’s role
BC Practitioner Confidentiality Extension

Indicates the confidentality of the practitioner’s information.
BC Practitioner Disciplinary Actions

Provides details of disciplinary actions against the provider.
BC Practitioner Qualification Issue Date

The issue date of the practitioner’s qualification.
BC Practitioner to Practitioner Relationship

Allows for relationships of practitioners to practitioners without needing CareTeam semantics.
BC Status Extension

Tracking status codes and the effective dates of those statuses.
BC Valid Flag Extension

Used to indicate if a data element has been validated by the ministry.

Terminology: Value Sets

These define sets of codes used by systems conforming to this implementation guide

BC Valid Flag Extension ValueSet

An indication of the validity of a data element

Terminology: Code Systems

These define new code systems used by systems conforming to this implementation guide

BC Valid Flag Extension CodeSystem

An indication of the validity of a data element

Example: Example Instances

These are example instances that show what data produced and consumed by systems conforming with this implementation guide might look like

Example-AddOrganization-Bundle

Example of a Bundle of resources that are needed to create an Organization.
Example-AddOrganization-Location

Example of a Location (Facility) that exists on its own.
Example-AddOrganization-Organization

Example of an Organization that is being created.
Example-AddOrganization-OrganizationAffiliation

Example of a relationship between an Organization and its Locations (facilities).
Example-AddProvider-Bundle

Example of a bundle of resources sent when requesting a provider create.
Example-AddProvider-Practitioner

Example of a BC practitioner that is being created.
Example-AddProvider-PractitionerRole

Example of the role that the created practitioner is playing.
Example-AddProvider-RelatedPractitioner

Example of a practitioner that has a relationship to the example created practitioner.
Example-AddProvider-WorkLocation

Example of a Location (facility) that is the affiliated with an Organization.
Example-Qualification-Organization

Example of an organization’s qualifications.
